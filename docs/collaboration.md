# 協作指南

## 🎯 **協作概述**

本文檔定義了夥伴 A 和夥伴 B 之間的協作流程、溝通方式和責任劃分。

---

## 👥 **團隊成員**

### **夥伴 A (Web 模組負責人)**
- **主要職責**: 網站前端、用戶介面、儀表板功能
- **技術棧**: Flask, HTML/CSS/JavaScript, Bootstrap
- **負責模組**: `/web_modules/`

### **夥伴 B (AI 模組負責人)**
- **主要職責**: AI 功能、LINE Bot、智慧客服
- **技術棧**: Google Gemini API, LINE Messaging API, Python
- **負責模組**: `/ai_modules/`

---

## 🔄 **協作流程**

### **1. 開發流程**
```
1. 需求討論 → 2. API 設計 → 3. 並行開發 → 4. 集成測試 → 5. 部署上線
```

#### **需求討論**
- 在 `/docs/collaboration.md` 中記錄新需求
- 雙方共同評估技術可行性
- 確定功能範圍和完成時間

#### **API 設計**
- 根據需求設計 API 介面
- 在 `/docs/api_specs.md` 中定義規格
- 雙方確認 API 格式和參數

#### **並行開發**
- 夥伴 A 開發 Web 端
- 夥伴 B 開發 AI 端
- 通過 API 介面進行協作

#### **集成測試**
- 測試 API 介面是否正常
- 驗證數據傳遞是否正確
- 確認功能完整性

#### **部署上線**
- 同步部署到生產環境
- 監控系統運行狀況
- 收集用戶反饋

### **2. 溝通機制**

#### **日常溝通**
- **即時溝通**: 使用團隊聊天工具
- **正式溝通**: 在 `/docs/collaboration.md` 中記錄
- **技術討論**: 使用 GitHub Issues 或類似工具

#### **會議安排**
- **每週例會**: 討論進度和問題
- **需求評審**: 評估新需求的可行性
- **技術分享**: 分享技術心得和最佳實踐

#### **文檔更新**
- **API 變更**: 立即更新 `/docs/api_specs.md`
- **功能更新**: 更新相關的技術文檔
- **問題記錄**: 記錄解決過程和方案

---

## 🚫 **禁止行為**

### **嚴格禁止**
- ❌ **直接修改對方模組**: 絕對不能直接修改對方的程式碼
- ❌ **繞過 API 介面**: 不能直接調用對方的內部函數
- ❌ **修改共享結構**: 不能單方面修改 `/shared/models.py` 和 `/shared/database.py`
- ❌ **獨立部署**: 不能單方面部署到生產環境

### **需要協商**
- ⚠️ **API 格式變更**: 需要雙方同意
- ⚠️ **資料庫結構修改**: 需要評估影響範圍
- ⚠️ **重大功能變更**: 需要充分測試
- ⚠️ **第三方服務變更**: 需要評估依賴關係

---

## ✅ **允許行為**

### **協作區域**
- ✅ **API 介面調用**: 可以調用對方提供的 API
- ✅ **共享工具函數**: 可以在 `/shared/utils.py` 中新增工具函數
- ✅ **文檔編寫**: 可以更新和編寫技術文檔
- ✅ **測試代碼**: 可以編寫和運行測試

### **溝通方式**
- ✅ **技術討論**: 討論技術實現方案
- ✅ **問題反饋**: 報告發現的問題和建議
- ✅ **進度同步**: 同步開發進度和狀況
- ✅ **知識分享**: 分享技術經驗和學習資源

## 🐛 **問題處理**

### **1. 衝突解決**
#### **檔案衝突**
- **發現**: Git 合併時發現衝突
- **處理**: 立即通知對方，共同解決
- **預防**: 經常同步代碼，避免長時間分支

#### **API 衝突**
- **發現**: API 呼叫失敗或數據格式不匹配
- **處理**: 檢查 API 規格，確認雙方實現一致
- **預防**: 使用版本控制，明確標註變更

#### **功能衝突**
- **發現**: 功能重複或邏輯衝突
- **處理**: 在 `/docs/collaboration.md` 中記錄，討論解決方案
- **預防**: 開發前充分溝通，明確功能邊界

### **2. 緊急處理**
#### **生產環境問題**
- **立即通知**: 發現問題立即通知對方
- **快速診斷**: 確定問題來源和影響範圍
- **協同修復**: 按照責任劃分進行修復
- **事後檢討**: 分析問題原因，預防再次發生

#### **服務中斷**
- **等級分類**: 根據影響範圍確定處理優先級
- **溝通協調**: 協調修復資源和時間安排
- **用戶通知**: 及時通知用戶服務狀況
- **恢復驗證**: 確認問題完全解決
